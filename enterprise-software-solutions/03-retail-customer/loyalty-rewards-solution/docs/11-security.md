# 11 - Loyalty Security & Fraud Prevention

## Point Integrity
- **Transactional Ledgers**: Every point change must have a corresponding "Reason Code" and "Transaction ID."
- **Double-Spend Protection**: Using distributed locks during the redemption phase.

## Fraud Detection
- **Anomaly Detection**: Flagging accounts with sudden, uncharacteristic point spikes.
- **Velocity Tracking**: Limiting the number of redemptions allowed per day/hour.

## Privacy & Compliance
- **GDPR**: Allowing users to export their loyalty data or delete their profile.
- **Encryption**: sensitive PII (emails/phones) hashed during storage for analytics.
