# 09 - Delivery System Workflows

## The Delivery Lifecycle
1. **Manifesting**: Orders arrive from the E-commerce system.
2. **Optimization**: Dispatcher initiates "Route Planning"; System sequences stops for maximum efficiency.
3. **Dispatch**: Tasks pushed to driver apps; Manifest confirmed by drivers.
4. **Transit**: Driver moves from stop to stop; Live ETAs shared with customers.
5. **Execution**: Driver arrives at geofenced location; App prompts for ePOD.
6. **Closing**: ePOD captured; Shipment marked "Delivered"; Final notification sent to customer.

## Exception Workflow
1. Driver flags a stop as "Undeliverable" (e.g., No access/Customer not home).
2. Dispatcher receives real-time alert with driver comments/photo.
3. System automatically attempts to reschedule or returns order to warehouse hub.
