## Step-by-Step Documentation Flow

1. Define the product vision in **01-overview.md**.
2. Break the platform into business capabilities in **02-features.md**.
3. Convert capabilities into services and bounded modules in **03-modules.md**.
4. Map permissions and operating responsibilities in **04-user-roles.md**.
5. Choose the architecture in **05-system-architecture.md**.
6. Design the domain model in **06-database-design.md**.
7. Define integration contracts in **07-api-design.md**.
8. Describe user journeys in **08-ui-pages.md** and **09-workflow.md**.
9. Lock the implementation choices in **10-tech-stack.md** through **12-deployment.md**.
10. Use **13-future-features.md** to plan expansion.
11. Use **14-development-prompts.md** and **15-task-status.md** to execute delivery.
