# 11 - ERP Security & Compliance

## Data Integrity & Safety
- **ACID Transactions**: Ensuring that a financial transaction is either 100% complete or never started, preventing skewed books.
- **Row-Level Security (RLS)**: Restricting data access based on department or geographic region.

## Compliance
- **Audit Trails**: Non-repudiation of transactions; Logging which user approved every PO or Ledger adjustment.
- **SOX/GDPR/CCPA**: Pre-built tools for managing data sovereignty and internal financial controls.

## Infrastructure Security
- **HSM (Hardware Security Modules)**: For securing enterprise-level cryptographic keys.
- **IP White-listing**: Restricting administrative access to corporate VPNs or specific office ranges.
